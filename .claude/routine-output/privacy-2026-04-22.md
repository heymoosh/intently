## Memory / prompt content check (rules 2 + 4)

- `markdown_files` (raw journal, daily log, weekly goals): permitted by the privacy policy data-class table — "Storage layer (encrypted), transient prompt context with necessity justification." Necessity is documented in skill prompts (weekly-review step 0 needs the journal to surface emotional threads; this is the legitimate-raw-content edge case the routine spec explicitly says not to flag).
- `conversations.turns` (raw chat): permitted in storage with necessity (daily-review chat reconciliation), but **retention is unbounded** — see MEDIUM finding #3.
- `life_ops_config` JSONB: configuration only (schedule times, paths, integration flags). Not personal content.
- `projects`, `events`: structural metadata + titles. Acceptable per the data-class table.
- Skill prompts (`agents/*/SKILL.md`): contain no embedded user data. User data enters via tool calls scoped by `user_id`.

## Consent / deletion check (rule 5)

- Consent: V1 single-user (Muxin dogfoods); implicit per privacy policy. `agents/setup/SKILL.md` step 1.5 explicitly elicits OAuth consent for calendar/email and respects skip. **OK for V1.**
- Deletion: `on delete cascade` from `auth.users` removes all per-user rows on account deletion (verified across all 6 tables in `0001_initial_schema.sql`). No per-record soft-delete, no 30-day window logic. **Acceptable for V1** — UI exposes no delete affordance yet — but a deletion strategy must land before multi-user.
